# System of Record Template

Use this as your canonical property source of truth.

This structure mirrors the same operating lanes used in the live Balay workflow: profile, operations scripts, listings parity, media, and controlled change history.

## Why this exists

The SoR keeps website content, OTA listings, staff operations, and agent responses aligned.

If this is weak, everything downstream drifts.

## What to maintain here

- Property profile (amenities, rules, policies, location, rates)
- Guest operations scripts and workflows
- Listing copy parity (website, Airbnb, Booking)
- Media references (photos and videos)
- Change history
