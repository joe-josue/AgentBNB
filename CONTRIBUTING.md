# Contributing

Thanks for your interest in AgentBNB.

AgentBNB is an OSS extraction of a working hospitality operations stack. The source pattern is Balay Pansol with Gideon, an OpenClaw-powered hospitality agent.

## Scope

- Keep contributions public-safe and clean-room.
- Do not include private Balay Pansol operations data, credentials, customer details, family details, staff details, or live sheet IDs.
- Keep examples generic and reusable across properties.
- Prefer changes grounded in the working Balay/Gideon flow over speculative features.

## Good Contributions

- Improve setup clarity for the stack templates.
- Generalize a real production behavior into a reusable template.
- Improve owner approval, guest communication, or staff handoff safety.
- Clarify source-of-record contracts.
- Remove property-specific assumptions from reusable examples.

## How To Contribute

1. Open an issue describing the problem or improvement.
2. Keep pull requests focused to one capability or documentation lane.
3. For feature PRs, include:
   - what changed
   - why it matters for operators
   - any migration or setup notes

## Pull Request Checklist

- [ ] The change is reusable beyond Balay Pansol.
- [ ] The change is grounded in a real workflow or clearly documented operator need.
- [ ] No secrets, guest records, or private property data are included.
- [ ] Sensitive actions still require owner approval.
- [ ] Docs or templates are updated when contracts change.

## Code Of Conduct

Be respectful, practical, and implementation-focused.
